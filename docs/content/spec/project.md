# Project

The Project section describes the software system represented by a TechSheet.

It provides identifying and organizational metadata used for governance, ownership mapping, portfolio analysis,
reporting, and cross-system correlation.

Unlike Technologies, which describe technical components within a system, the Project section describes the system
itself.

The Project section is optional. A TechSheet may describe technologies without explicitly defining project metadata.

## Typical Information

Project metadata commonly includes:

* stable project identifiers
* human-readable names
* descriptive context
* maintaining teams or ownership structures
* repository and documentation references

## Governance Concerns

Ownership clarity, accountability, portfolio visibility, lifecycle management, and organizational reporting

## Classification Rule

Metadata describing the software system itself belongs to the Project section rather than a Technology classification.
