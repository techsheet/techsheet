**A full-stack [PHP](../language/php.md) framework built on an HMVC architecture, designed for flexibility and
community-driven development.**

FuelPHP is a web application framework for [PHP](../language/php.md) that implements a Hierarchical
Model-View-Controller (HMVC) pattern,
allowing modular organization of application code through nested MVC triads. It was created in 2011 by developers who
had previously contributed to [CodeIgniter](codeigniter.md), aiming to address perceived limitations
in that framework's
architecture and extensibility. FuelPHP provides an object-oriented codebase, built-in ORM (based on the Active Record
and Data Mapper patterns), input filtering, CSRF protection, and a command-line utility called Oil for code generation,
database migrations, and task execution.

The framework supports multiple template parsers, RESTful controller implementations, and a modular package system
managed through [Composer](../tool/composer.md). FuelPHP gained adoption in the early 2010s as a modern
alternative to
[CodeIgniter](codeigniter.md) before the rise of [Laravel](laravel.md), which
eventually captured a larger share of
the PHP framework ecosystem. Development activity on FuelPHP has slowed significantly since its 1.x series, and a
planned 2.0 rewrite was never completed, leaving it primarily in maintenance mode for existing projects.
