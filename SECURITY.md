# Security Policy

## Supported Versions

The two most recent major releases receive security updates. Non-critical patches are only applied to the latest major
release.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities by email to **hello@techsheet.org**. Include a description of the issue, the steps required to
reproduce it, and — where possible — a suggested fix or mitigation.

You can expect an initial response within **72 hours**. If the vulnerability is confirmed, a patched release will be
prepared as quickly as possible and you will be credited in the release notes (unless you prefer to remain anonymous).
If the report is declined, you will receive an explanation of why the issue is considered out of scope or not a security
risk.
